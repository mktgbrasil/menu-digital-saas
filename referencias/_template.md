# Template de entrada detalhada

Para referências que merecem mais que uma linha na tabela (ex.: um repo que a
gente vai realmente integrar, ou uma skill que vamos construir), crie um arquivo
`referencias/detalhes/<slug>.md` copiando o modelo abaixo.

---

## <Título curto da referência>

- **Categoria:** 🧩 skill / 💻 código / 🎨 site / 💡 ideia
- **Status:** 🆕 a triar / 🔍 avaliar / 🔨 executar / ✅ feito / 🗄️ arquivo
- **Link original:** <url do reel / repo / site>
- **Data salva:** AAAA-MM-DD

### O que é
<Resumo em 1–3 frases do que o reel/referência mostra.>

### Por que pode nos ajudar
<Onde isso encaixa no menu-digital-saas ou no nosso jeito de trabalhar.>

### Transcrição / análise (saída do /watch)
<Cole aqui o resultado da skill claude-video, se houver.>

### Próximo passo
- [ ] <ação concreta, se o status for 🔨 executar>
